# Technical Architecture Reference

This directory contains the technical reference documentation for WebInvestigator.

Unlike the [Architecture Handbook](../handbook/README.md), which explains architectural concepts, this directory describes the current implementation of the system.

It serves as a reference for developers working on the codebase.

---

## Purpose

This directory contains technical documentation such as:

- architecture diagrams,
- C4 diagrams,
- UML diagrams,
- sequence diagrams,
- dependency graphs,
- execution flow documentation,
- module interaction diagrams,
- other implementation-oriented documentation.

The documents in this directory describe the architecture as it currently exists.

---

## Relationship with Other Documentation

WebInvestigator separates architectural documentation into three complementary collections.

| Documentation | Purpose |
|--------------|---------|
| **[Project README](../../README.md)** | Introduces the project |
| **[Architecture Handbook](../handbook/README.md)** | Explains the architectural philosophy |
| **[Architectural Decision Records](../adr/README.md)** | Explains why architectural decisions were made |
| **Technical Architecture Reference** | Describes the current implementation |

Each collection has a distinct responsibility.

---

## Evolution

The Technical Architecture Reference is intended to evolve alongside the implementation.

Unlike the [Architecture Handbook](../handbook/README.md), which captures enduring architectural principles, this collection should always reflect the current state of the architecture.
