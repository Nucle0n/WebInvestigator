# Technical Architecture Reference

This directory contains the technical reference documentation for WebInvestigator.

Unlike the Architecture Handbook, which explains architectural concepts, this directory describes the current implementation of the system.

It serves as a reference for developers working on the codebase.

---

## Purpose

This directory contains technical documentation such as:

- architecture diagrams,
- C4 diagrams,
- UML diagrams,
- sequence diagrams,
- dependency graphs,
- execution flow documentation,
- module interaction diagrams,
- other implementation-oriented documentation.

The documents in this directory describe the architecture as it currently exists.

---

## Relationship with Other Documentation

WebInvestigator separates architectural documentation into three complementary collections.

| Documentation | Purpose |
|--------------|---------|
| **README** | Introduces the project |
| **Architecture Handbook** | Explains the architectural philosophy |
| **Architectural Decision Records** | Explains why architectural decisions were made |
| **Architecture Reference** | Describes the current implementation |

Each collection has a distinct responsibility.

---

## Evolution

The content of this directory is expected to evolve alongside the implementation.

Unlike the handbook, which changes infrequently, the technical reference should always reflect the current state of the architecture.
